# Installation Overview

The fastest way to get started is **Betterlytics Cloud**: create an account, add one script tag, and see your first data within minutes. No servers to run, no updates to manage, and always the latest features.

[**Get started in under a minute →**](/installation/cloud-hosting)

Prefer to run everything on your own infrastructure? Betterlytics is open source and can be [self-hosted with Docker](/installation/self-hosting).

## Cloud vs Self-Hosted

Both options provide the same privacy-first, cookieless analytics experience. Cloud is fully managed and always up to date, while self-hosting gives you full control over your data and infrastructure.

<HostingComparisonTable />

**How to read the table:**

- **Configurable** - Included in the self-hosted image, but requires configuration on your side (for example SMTP credentials for email features).
- A cross in the self-hosted column does not mean never. Most features are planned for self-hosting and arrive with later image updates, while some depend on additional infrastructure first (for example, session replays require S3-compatible object storage).

New features launch on Betterlytics Cloud first and reach the self-hosted image with periodic updates. See [feature availability for self-hosting](/installation/self-hosting#feature-availability) for how this works.
