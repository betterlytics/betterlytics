# Introduction

**Betterlytics** is a privacy-first platform that combines **web analytics**, **error tracking**, **session replay**, and **uptime monitoring** in a single dashboard. Instead of running one tool for analytics, another for error tracking, a third for session replay, and a fourth for uptime monitoring, you get all of them with one lightweight script and one dashboard.

Everything is cookieless by design and ready for GDPR, CCPA, and PECR out of the box, with no cookie banner required. The core tracking script is under 2KB, loads asynchronously, and optional features only load when you enable them.

You can use Betterlytics as a [managed cloud service](/installation/cloud-hosting) or [self-host](/installation/self-hosting) it on your own infrastructure. See the [Installation Overview](/installation) for a full comparison.

## One Platform, Every Signal

### Web Analytics

Understand your traffic and audience: visitors, pageviews, bounce rate, scroll depth, visit duration, [referrers and channels](/dashboard/referrers), [UTM campaigns](/dashboard/campaigns), [geography](/dashboard/geography), and [device breakdown](/dashboard/devices). Go deeper with [user journeys](/dashboard/user-journey), [conversion funnels](/dashboard/funnels), [custom events](/integration/custom-events), and [outbound link tracking](/dashboard/outbound-links).

### Error Tracking

Catch client-side JavaScript errors as they happen. Identical errors are [grouped automatically](/dashboard/errors) so you can see which issues affect the most users, inspect stack traces, and follow the session trail that led to the problem. No separate error tracking service needed.

### Session Replay

[Replay real visitor sessions](/dashboard/session-replay) with event timelines and privacy-first masking: all text and inputs are masked by default. Jump from an error straight to the replay of the moment it happened.

### Uptime Monitoring

[Monitor your website's availability](/dashboard/monitoring), response times, and SSL certificates with automated checks and incident tracking. Get alerted by email or via [Slack, Discord, Teams, Pushover, or webhooks](/dashboard/integrations) when something goes wrong.

### Performance

Track [Core Web Vitals](/dashboard/web-vitals) (LCP, INP, CLS, TTFB) from real users to see how your site actually performs for your visitors.

All data is available in real time through the [dashboard](/dashboard), with [filtering](/dashboard/filtering) across any dimension. You can also receive [scheduled email reports](/dashboard/email-reports) summarizing your key analytics, and query everything from your AI assistant via the built-in [MCP server](/dashboard/mcp).

## Privacy Without Compromise

Betterlytics collects no personal data and uses no cookies, so there is nothing to consent to for core analytics. Visitor IP addresses are anonymized, and each website's data is isolated to prevent cross-site tracking. Optional features that could touch personal data, like Session Replay, are disabled by default, mask everything out of the box, and support explicit consent flows. See the [FAQ](/faq) for details on compliance.
